# Security

Si tu trouves une faille de securite ou un comportement qui expose des donnees financieres:

- n'ouvre pas d'issue publique avec les details sensibles
- prepare un rapport minimal avec impact, reproduction et version concernee
- partage le correctif ou les details au mainteneur du repo en prive

## Bonnes pratiques pour les contributors

- ne committe jamais de base SQLite reelle
- ne committe jamais de CSV bancaire personnel
- change les credentials demo avant tout deploiement public
- utilise HTTPS et un secret de session fort sur une instance hebergee
